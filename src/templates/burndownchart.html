Burndown chart

Frequentemente usado no scrum, um gráfico de burndown é um gráfico exibido publicamente mostrando o trabalho restante. Atualizado todos os dias, ele fornece visualizações rápidas para referência, mostrando uma comparação entre o real e o previsto.

Funcionalidade do Burndown chart:

No gráfico de burndown, traçamos o volume estimado do trabalho que precisa ser feito em relação ao tempo necessário para concluir este trabalho. O objetivo é mostrar uma distribuição precisa do tempo e facilitar o planejamento.

Guia: Exemplo Burndown chart

Este guia oferece para você o passo-a-passo essencial de como estruturar o Burndown chart.

Guia: Exemplo do Burndown chart

  1. Defina a escala de tempo: Escolha a unidade de tempo (dias ou semanas) para representar o progresso do Sprint.

    Estabeleça uma linha de base: Determine o ponto de partida, geralmente a quantidade total de trabalho planejado no primeiro dia do Sprint.

    2. Liste as tarefas: O Burndown Chart lista tarefas no eixo vertical (trabalho) e deve ser atualizado diariamente com o trabalho restante à medida que as tarefas são concluídas.

    Inclua pontos de controle: Adicione marcos ou linhas de referência que representam os objetivos diários de progresso.

    Identifique desvios e compartilhe: O Burndown Chart monitora o progresso da equipe, detecta desvios e compartilha informações para transparência e melhorias contínuas.

    Dica: Utilizando o Jira( gerenciador de processos) e atualizando todos os dias, o gráfico burndown chart é gerado automaticamente.

Leitura do gráfico Burndown Chart:
  • Descrição: A linha vertical representa os valores dos pesos das tarefas x e a linha horizontal representa o tempo para a realização das tarefas (tempo total da sprint). Atualizado a produção diariamente, ele fornece visualizações rápidas para referência. A linha diagonal vermelha é o tempo ideal de produção x tempo. A linha azul mede o desempenho de produção x tempo. Quando a linha azul está abaixo da diagonal significa que o projeto esta adiantado. Caso esteja acima da diagonal o projeto esta atrasado referente a produtividade ideal.
